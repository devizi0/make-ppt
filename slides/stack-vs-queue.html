Stack — 후입선출(LIFO)

📦
Last In, First Out
마지막에 넣은 데이터가 가장 먼저 나온다 — 책 더미를 떠올려라
⬆️
push / pop 연산
push로 위에 쌓고, pop으로 위에서 꺼낸다. 양쪽 끝 접근은 불가
⚡
O(1) 시간복잡도
push·pop·peek 모두 상수 시간 — 삽입·삭제가 빠르다
🔄
재귀적 특성
함수 호출 스택과 동일한 구조 — 재귀 알고리즘과 자연스럽게 맞닿아 있다

Queue — 선입선출(FIFO)

🚶
First In, First Out
먼저 들어온 데이터가 먼저 나간다 — 줄 서기와 동일한 원리
➡️
enqueue / dequeue 연산
rear에서 삽입(enqueue), front에서 제거(dequeue) — 단방향 흐름
⚡
O(1) 시간복잡도
enqueue·dequeue 모두 상수 시간. 연결 리스트 구현 시 최적
🔁
순서 보장
입력 순서를 그대로 유지 — 공정한 처리가 필요한 곳에 적합

실전 활용 사례

🔙 브라우저 뒤로가기 Stack
방문 페이지를 스택에 push, 뒤로가기 시 pop — 방문 이력을 역순으로 탐색
↩️ Undo / Redo Stack
편집 작업을 스택에 기록, Ctrl+Z 시 pop으로 이전 상태 복원
🧮 함수 호출 스택 Stack
함수 호출마다 스택 프레임 push, 반환 시 pop — OS가 실제로 사용하는 방식
🖨️ 프린터 대기열 Queue
먼저 요청한 문서가 먼저 출력 — 공정한 순서 처리의 대표 사례
🌐 BFS 탐색 Queue
그래프 너비 우선 탐색에서 방문 노드를 큐에 enqueue, 순서대로 처리
📨 메시지 큐 Queue
Kafka·RabbitMQ 등 비동기 처리 시스템 — 순서 보장이 핵심
