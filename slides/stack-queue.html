스택 (Stack)

📚

LIFO — Last In, First Out

마지막에 들어온 데이터가 가장 먼저 나간다. 책을 쌓고 꺼내는 것처럼 위에서만 접근한다.

⬆️

핵심 연산: push / pop

push: 스택 맨 위에 데이터 추가  |  pop: 맨 위 데이터 제거 및 반환

👁️

단일 접근점 — top만

peek으로 제거 없이 최상단 확인. 중간 데이터에는 직접 접근 불가 — 반드시 순차 pop.

Stack 구조
↑ push / pop ↑
D  ← top
C
B
A  bottom
한쪽(top)에서만 삽입·삭제

큐 (Queue)

🎟️

FIFO — First In, First Out

먼저 들어온 데이터가 가장 먼저 나간다. 줄을 서는 것처럼 앞에서 나가고 뒤에서 들어온다.

➡️

핵심 연산: enqueue / dequeue

enqueue: rear(뒤)에 데이터 추가  |  dequeue: front(앞)에서 데이터 제거 및 반환

↔️

양방향 접근 — front & rear

삽입과 삭제가 서로 반대쪽에서 일어난다. front는 나가는 곳, rear는 들어오는 곳.

Queue

프린터 대기열

인쇄 요청을 enqueue, 프린터는 순서대로 dequeue하여 먼저 요청한 작업부터 처리
